先进行数据处理

  • 一般实际的数据很少没有缺失值的，所以要考虑把缺失值补上；

  • 决策树训练需要特征值，也就是将特征值处理出来，有些字段不仅很难处理成特征值而且即使处理后也对训练模型没有多大帮助；

  • 决策树的特征值只能是小数或者是整数，因此字符串的特征值需要进行映射处理

以上面数据为例：

<font size="4">

  1. Name 这个特征很杂很散，而且是字符串，很难映射成数值，因此抛弃掉。 <br/>
  2. Sex 这个特征值虽然是字符串，但只有两种，很好映射，因此保留并映射成 0 和 1。<br/>
  3. Age 这个特征值很明显有空缺，然后根据大佬推荐用其他 Age 的平均值补上空缺。 <br/>
  4. SibSp 和 Parch 这两个特征值只有数字，直接取出保留。 <br/>
  5. Ticket 问题和 Name 一样，抛弃。 <br/>
  6. Fare 均为小数，但是也有缺失值的问题，采用均值补上 <br/>
  7. Cabin 数据缺失值很多，而且也是字符串型的特征值，舍弃 <br/>
  8. Embarked 只有 C,S,Q 三种字符，可以映射后当做特征值，保留 <br/>
    </font>

这是初步的想法，在KaggleKaggleKaggle 上看了一下别人的数据处理方式，深有体会，别人的处理方法可以保留更多信息，下面是他们的处理方法

<font size="4">

  1. Name 这个特征虽然很复杂，但是英文名往往会有头衔，如 Mr、Mrs、Master 等，可以抽象出来成一个特征。 <br/>
  2. Sex 这个特征值虽然是字符串，但只有两种，很好映射，因此保留并映射成 0 和 1。<br/>
  3. Age 这个特征值很明显有空缺，用平均值补全。 <br/>
  4. SibSp 和 Parch 这两个特征值只有数字，将这两个特征值加起来成为一个 FamilySize 特征，再根据这个 FamilySize 特征可以抽取出 IsAlone 特征，也就是如果 FamilySize 是 1 的话就是独身一人，IsAlone 为 1。 <br/>
  5. Ticket 确实没办法处理，抛弃掉。 <br/>
  6. Fare 均为小数，但是也有缺失值的问题，采用均值补上 <br/>
  7. Cabin 数据缺失值很多，但我们可以抽取出 HasCabin 特征，如果有，则为 1，否则为 0 <br/>
  8. Embarked 只有 C,S,Q 三种字符，可以映射后当做特征值，保留 <br/>
    </font>
